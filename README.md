# All2API Desktop

All2API Desktop 是一个基于 `Vue 3`、`Vite`、`Tailwind CSS` 和 `Electron` 的桌面端账号自动化管理工具。当前重点支持 Kiro / AWS Builder ID 注册链路，提供账号创建、邮箱接码、浏览器流程预览、指纹注入和本地账号落盘能力。

## 功能概览

### 账号注册与管理
- Kiro 登录页自动选择 `Builder ID` 注册入口。
- AWS Builder ID 注册流程自动填写邮箱、姓名、验证码和密码。
- 支持临时邮箱自动创建与验证码轮询。
- 支持随机浏览器指纹注入，降低批量流程中的环境重复度。
- 注册过程提供实时日志和浏览器截图预览。
- 成功注册后记录账号、邮箱、姓名、密码和 token 预览。
- 账号数据同时保存到应用状态和本地 JSON 文件。

### 账号中心界面
- **响应式卡片布局**：账号卡片固定宽度 280px，根据屏幕尺寸自动调整每行显示数量，充分利用大屏幕空间。
- **账号快速切换**：一键切换到指定 Kiro 账号，自动启动 Kiro 客户端并登录，当前活跃账号有明确的视觉标识。
- **反代配置管理**：为每个账号单独配置代理服务器，支持 HTTP/HTTPS/SOCKS5 协议，支持代理认证。
- **详细额度信息**：显示套餐额度、免费额度和福利额度的详细使用情况，提供分段进度条和额度刷新功能。

## Kiro 注册流程

当前自动化流程按真实页面顺序执行：

1. 打开 `https://app.kiro.dev/signin`。
2. 点击 Kiro 登录方式中的 `Builder ID`。
3. 进入 AWS Builder ID 页面，填写邮箱并点击 `Continue`。
4. 填写随机姓名，等待 5 秒后点击 `Continue`。
5. 从邮箱获取 6 位验证码，填写后点击 `Continue`。
6. 随机生成包含数字、大写字母、小写字母的密码，并填写两次。
7. 根据页面提示完成授权确认与 `Allow access`。
8. 等待 Kiro 授权 Cookie 稳定后写入账号记录。

## 本地账号文件

成功注册的账号会写入 Electron 应用数据目录：

```text
~/Library/Application Support/All2API/accounts.json
```

记录字段包括：

```json
{
  "providerId": "kiro",
  "email": "example@domain.com",
  "username": "kiro_...",
  "fullName": "Alex Chen",
  "password": "Abc123...",
  "ssoCookieName": "x-amz-sso_authn",
  "ssoTokenPreview": "...",
  "createdAt": "2026-04-28T00:00:00.000Z"
}
```

应用界面仍会使用 `localStorage` 保存前端状态；本地 JSON 文件用于长期留存账号密码。

## 技术栈

- `Electron`：桌面应用外壳与主进程自动化能力。
- `Vue 3`：前端界面。
- `Pinia`：账号中心和配置状态管理。
- `Vite`：开发和构建工具。
- `Tailwind CSS`：界面样式。
- `Playwright`：浏览器自动化注册流程。

## 环境要求

- Node.js 18 或更高版本。
- npm。
- 可访问 Kiro、AWS Builder ID 和所选临时邮箱服务的网络环境。

## 注意事项

- 请只在你有权限的网站和账号体系中使用本工具。
- 临时邮箱服务可能受网络、频率和服务可用性影响。
- 账号密码会以明文写入本地 JSON 文件，请注意本机文件权限和数据安全。
- AWS / Kiro 页面结构可能调整，如果选择器失效，需要同步更新自动化流程。

## 账号中心功能详解

### 响应式卡片布局

账号卡片采用固定宽度（280px）的响应式布局，根据窗口大小自动调整每行显示的卡片数量：

- **980px 宽度**（最小窗口）：约 3 个卡片/行
- **1280px 宽度**：约 4 个卡片/行
- **1920px 宽度**：约 6 个卡片/行
- **2560px 宽度**：约 8 个卡片/行

卡片之间保持 12px 的间距，最后一行卡片左对齐，不会被拉伸填充。窗口大小调整时，布局会自动重排。

### 账号快速切换

在每个 Kiro 账号卡片的右上角，提供了快速切换按钮：

1. **切换账号**：点击切换按钮后，应用会自动启动 Kiro 桌面客户端并切换到该账号。
2. **活跃账号标识**：最近 30 分钟内启动过的账号会被标记为活跃账号，其切换按钮显示为禁用状态。
3. **切换状态反馈**：切换过程中会显示加载动画，成功或失败后会显示相应的提示信息。

**注意**：
- 切换功能仅在 Electron 桌面环境下可用。
- 需要确保 Kiro 客户端已正确安装并配置路径。

### 反代配置

为每个账号提供独立的代理服务器配置：

#### 配置步骤

1. 点击账号卡片右上角的**反代按钮**（齿轮图标）。
2. 在弹出的配置面板中填写以下信息：
   - **启用/禁用开关**：控制是否使用代理
   - **服务器地址**：代理服务器的地址（必填）
   - **端口号**：代理服务器的端口（1-65535，必填）
   - **协议类型**：选择 HTTP、HTTPS 或 SOCKS5
   - **用户名**：代理认证用户名（可选）
   - **密码**：代理认证密码（可选）
3. 点击**保存**按钮保存配置。

#### 配置说明

- **已配置标识**：已配置代理的账号，反代按钮会显示为绿色高亮。
- **配置持久化**：代理配置会自动保存到本地存储，下次打开应用时仍然有效。
- **独立配置**：每个账号可以配置不同的代理服务器，互不影响。

#### 使用场景

- 在不同网络环境下使用不同的代理策略
- 为特定账号配置专用代理通道
- 绕过网络限制访问 Kiro 服务

### 额度信息展示

账号卡片显示详细的额度使用情况：

#### 额度类型

- **套餐额度**：付费订阅获得的配额（青色标识）
- **免费额度**：免费提供的基础配额（绿色标识）
- **福利额度**：促销或奖励获得的额外配额（紫色标识）

#### 显示内容

- **总额度**：显示总已用/总额度
- **分段进度条**：使用不同颜色的进度条段显示各类额度的使用比例
- **详细列表**：列出每种额度类型的具体使用情况
- **警告标识**：当某类额度用尽时，显示 ⚠️ 警告图标

#### 额度刷新

点击额度信息旁边的**刷新按钮**可以手动同步最新的额度数据。刷新过程中会显示加载动画。

**注意**：
- 额度信息来源于 Kiro API 拦截或手动同步。
- 如果 API 不提供细分额度数据，则只显示总额度。
- 额度信息加载失败时会显示"待同步"状态。

## 常见问题

### Q: 为什么切换账号按钮是灰色的？

A: 这表示该账号是当前活跃账号（最近 30 分钟内启动过）。活跃账号无需再次切换，因此按钮被禁用。

### Q: 反代配置保存后不生效怎么办？

A: 请检查以下几点：
1. 确认代理服务器地址和端口号正确
2. 确认代理服务器可以正常访问
3. 如果使用了认证，确认用户名和密码正确
4. 尝试重启应用后再次测试

### Q: 额度信息显示"待同步"是什么意思？

A: 这表示应用还没有获取到该账号的额度信息。可能的原因：
1. 账号刚添加，还没有进行过 API 拦截
2. 额度信息同步失败
3. 该账号还没有登录过 Kiro Web 应用

解决方法：点击刷新按钮手动同步额度信息，或者在 Kiro Web 应用中使用该账号后再查看。

### Q: 如何配置 Kiro 客户端路径？

A: 在账号详情页面的设置区域，可以配置 Kiro 客户端的可执行文件路径和工作区路径。默认情况下，应用会尝试使用系统中已安装的 Kiro 客户端。

### Q: 反代配置的密码是否安全？

A: 反代配置数据存储在本地 localStorage 中。建议：
1. 不要在不受信任的设备上配置代理
2. 定期更换代理密码
3. 使用专用的代理账号，不要使用重要账号的密码

### Q: 响应式布局在我的屏幕上显示不正常怎么办？

A: 请确保：
1. 窗口宽度至少为 980px（最小支持宽度）
2. 浏览器缩放比例为 100%
3. 如果问题仍然存在，请尝试刷新页面或重启应用

### Q: 可以批量配置多个账号的代理吗？

A: 当前版本不支持批量配置。如果需要为多个账号配置相同的代理，需要逐个打开配置面板进行设置。批量操作功能计划在后续版本中添加。
